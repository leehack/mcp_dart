# MCP 2026-07-28 Draft/RC Transition Guide

`mcp_dart 2.3.0-dev.2` is dual-era: it implements the locked MCP `2026-07-28`
release-candidate core while retaining the MCP `2025-11-25` feature set and
earlier initialization compatibility. Its default `McpProtocol.stable` profile
prefers `server/discover` and stateless request metadata, then falls back for
legacy peers.

## Default client behavior

No protocol option is needed for the default 2026 negotiation:

```dart
final client = McpClient(
  const Implementation(name: 'my-client', version: '1.0.0'),
);
```

The default behavior is equivalent to
`McpClientOptions(protocol: McpProtocol.stable)`: it enables
`server/discover`, sends the `2026-07-28` draft/RC stateless request metadata,
and falls back to the legacy `initialize` flow when the peer looks like a
legacy MCP server.

Recognized modern protocol errors stay in modern negotiation: the client may
retry a mutually supported stateless version, but it does not reinterpret the
error as permission to run legacy `initialize`. Legacy-only server profiles
identify `server/discover` as unavailable so dual-era clients can fall back.

Use the legacy profile explicitly when a deployment must use only MCP
`2025-11-25` and earlier behavior:

```dart
final client = McpClient(
  const Implementation(name: 'my-client', version: '1.0.0'),
  options: const McpClientOptions(protocol: McpProtocol.legacy),
);
```

Use the strict profile for conformance tests or deployments where fallback is
not acceptable:

```dart
final client = McpClient(
  const Implementation(name: 'my-client', version: '1.0.0'),
  options: const McpClientOptions(protocol: McpProtocol.require2026),
);
```

## Default server behavior

`McpServerOptions()` also defaults to the stable profile. Servers advertise
and accept MCP `2026-07-28` draft/RC stateless requests alongside legacy
versions:

```dart
final server = McpServer(
  const Implementation(name: 'my-server', version: '1.0.0'),
);
```

Use `McpServerOptions(protocol: McpProtocol.legacy)` to advertise only legacy
MCP versions, or `McpServerOptions(protocol: McpProtocol.require2026)` to
reject legacy initialization.

## Profile summary

| Profile | Default? | Client behavior | Server behavior |
| ------- | -------- | --------------- | --------------- |
| `McpProtocol.stable` | Yes | Tries `server/discover`, then falls back to `initialize` | Advertises legacy and `2026-07-28` draft/RC protocol versions |
| `McpProtocol.legacy` | No | Uses legacy `initialize` | Advertises MCP `2025-11-25` and earlier protocol versions |
| `McpProtocol.require2026` | No | Requires `2026-07-28` draft/RC discovery | Advertises only stateless `2026-07-28` draft/RC protocol versions |

The `stable` profile name identifies the SDK's default compatibility policy;
it does not label the preferred wire version as final. The constants distinguish
that maturity explicitly: `stableProtocolVersion` is `2025-11-25`, while
`previewProtocolVersion` is `2026-07-28` for this prerelease.

## Low-level overrides

The existing low-level options remain available for advanced callers:

```dart
final client = McpClient(
  const Implementation(name: 'my-client', version: '1.0.0'),
  options: const McpClientOptions(
    protocolVersion: previewProtocolVersion,
    useServerDiscover: true,
  ),
);
```

Prefer the `protocol` profile unless you need to target a specific protocol
version for tests or interoperability debugging.

A supported legacy `protocolVersion` selects `initialize` directly unless
`useServerDiscover` is explicitly enabled. When discovery is forced, the probe
uses the latest stateless version and retains the requested legacy version only
for a genuine legacy fallback.

Use `previewProtocolVersion` to target MCP `2026-07-28` while the
upstream specification is in release-candidate validation. The earlier
`DRAFT-2026-v1` conformance alias is no longer exposed or accepted by the SDK.

## 2026-only protocol and extension APIs

The following features use MCP `2026-07-28` draft/RC behavior in this
branch's default stable profile. Select `McpProtocol.legacy` explicitly when a
caller must avoid draft behavior:

- `server/discover` negotiation and stateless per-request metadata.
- `subscriptions/listen` stateless notification streams, including
  `SubscriptionsListenResult` metadata on graceful stream closure.
- Multi-result tool/resource/prompt flows such as `input_required`.
- URL-mode `elicitation/create` uses the current draft schema shape with
  `mode`, `message`, and `url`; `elicitationId` and
  `notifications/elicitation/complete` are legacy/non-draft wire shapes.
- MCP Tasks extension flows using `io.modelcontextprotocol/tasks`.
- Non-object `structuredContent` values via `JsonValue` and broader server
  `outputJsonSchema` shapes.
- Stateless result metadata such as `resultType`, `ttlMs`, and `cacheScope`.

The RC retains request-scoped logging for compatibility but marks it
deprecated. New implementations should use `stderr` with stdio or OpenTelemetry
for structured observability; the current examples use progress and
`subscriptions/listen` instead.

For non-object tool results, keep the stable object-root APIs for stable MCP
callers and use the explicitly named draft APIs:

```dart
server.registerTool(
  'array-result',
  outputJsonSchema: JsonSchema.array(items: JsonSchema.string()),
  callback: (args, extra) {
    return CallToolResult.fromStructuredArray(['alpha', 'beta']);
  },
);

final result = await client.callTool(
  const CallToolRequest(name: 'array-result'),
);
final items = result.structuredContentJson?.asArray;
```

The draft/RC API surface may still change before the official spec release.
Use `McpProtocol.legacy` when an application cannot accept draft behavior;
otherwise the default `McpProtocol.stable` profile exercises 2026 with legacy
fallback.

## Release status

Published SDK and CLI prereleases provide the MCP `2026-07-28` preview channel.
The standalone install scripts track stable releases; activate an explicit CLI
prerelease when testing this protocol. Projects created by the dev.2 CLI use the
matching dev.2 SDK dependency.

[PR #306](https://github.com/leehack/mcp_dart/pull/306) merged the 2026
implementation line into `main`. The superseded RC PR is closed, while
`dev/2026-07-28-rc` remains a read-only archive for links embedded in the
earlier dev.0 and dev.1 packages.

CI runs official 2025 and 2026 conformance, pinned spec-example audits,
bidirectional TypeScript and Python interop, and a real Chrome smoke test. See
the [interoperability guide](interoperability.md#running-interop-checks-locally)
for local commands and the
[day-0 release runbook](mcp-2026-07-28-release-runbook.md) for the final tag,
metadata, publish, and public-verification gates.
